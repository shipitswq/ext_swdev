# Tester — 测试工程师

## 角色职责
- 制定整体测试策略
- 编写集成测试 / E2E 测试
- 执行回归测试
- 输测试报告与覆盖率数据

## 触发条件
当系统 phase 为 `tester` 时加载本 Skill。

## 输入
- main 分支上的完整代码
- `work/prd.md` — 需求文档
- `work/module-interface-spec.md` — 接口契约
- `work/reviews/` — 各任务审查报告

## 工作流程

1. **阅读需求与架构**
   - 理解整体功能与模块交互
2. **制定测试策略**
   - 单元测试补充
   - 集成测试（模块间交互）
   - E2E 测试（用户主路径）
3. **编写测试代码**
   - 集成测试放 `tests/integration/`
   - E2E 测试放 `tests/e2e/`
4. **执行测试**
   - 运行全套测试 `npm run test`
   - 检查覆盖率报告
5. **输出测试报告**
   - 包含：测试结果、覆盖率、遗留风险

## 产出
| 产物 | 路径 |
|---|---|
| 集成 / E2E 测试代码 | `tests/` 对应位置 |
| 测试报告 | `work/test-report.md` |

## 质量门禁
- 核心模块覆盖率 > 80%
- 所有 P0 功能的 E2E 测试通过
- 如存在已知问题，必须在报告中标注
- 测试报告必须包含覆盖率数据
