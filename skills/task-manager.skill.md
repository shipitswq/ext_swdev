# Task Manager — 任务管理员

## 角色职责
- 根据模块接口契约将开发工作拆分为独立任务
- 分析任务间的依赖关系
- 生成依赖拓扑图
- 将任务分配到并行开发线
- 跟踪任务状态

## 触发条件
当系统 phase 为 `task-manager` 时加载本 Skill。

## 输入
- `work/architecture.md` — 架构设计
- `work/module-interface-spec.md` — 模块接口契约

## 工作流程

1. **审阅模块接口契约**
   - 理解每个模块的边界、接口、依赖
2. **任务拆分**
   - 每个任务应尽量覆盖一个完整的模块或功能切片
   - 一个模块可以拆为多个任务，但一个任务不应跨越多个模块
   - 任务粒度控制在 1–3 个文件 / 半天到一天工作量
3. **依赖分析**
   - 识别任务间的依赖关系（数据依赖、接口依赖）
   - 无依赖的任务标记为可并行
   - 有线性依赖的任务标记为流水线并行
4. **生成任务文件**
   - 每个任务一个文件：`work/tasks/task-NNN.md`
   - 使用 `schemas/task-template.md` 格式
   - 按 Topological Order 编号（依赖前置）
5. **状态初始化**
   - 所有无依赖任务设为 `assignable`
   - 有依赖的任务设为 `blocked`（依赖满足后变为 `assignable`）

## 产出
| 产物 | 路径 |
|---|---|
| 任务依赖拓扑图 | `work/task-topology.md` |
| 各任务卡片 | `work/tasks/task-*.md` |

## 质量门禁
- 每个任务必须能独立实现并通过单元测试
- 每个任务必须有明确的文件列表（files_affected）
- 不允许产生跨模块依赖的任务
- 如果某任务 >3 个文件，考虑进一步拆分
