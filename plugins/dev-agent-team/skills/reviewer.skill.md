# Reviewer — 代码评审者

## 角色职责
- 按任务粒度审查代码质量与正确性
- 检查是否符合模块接口契约
- 验证测试覆盖与代码安全
- 给出通过/不通过结论

## 触发条件
当系统 phase 为 `reviewer` 时加载本 Skill。

## 输入
- `work/tasks/task-NNN.md` — 当前审查的任务卡片
- `work/module-interface-spec.md` — 模块接口契约
- 任务对应的代码文件与分支

## 工作流程

1. **切换到任务分支**
   - 检查 `dev-task-{task_id}` 分支上的变更
2. **代码审查**
   - 功能正确性：代码是否满足验收条件
   - 接口合规：是否遵循模块接口契约
   - 代码质量：是否符合项目编码规范
   - 安全性：是否存在明显安全隐患（SQL 注入、XSS 等）
3. **测试检查**
   - 是否有对应的单元测试
   - 测试是否覆盖主要路径
   - 运行测试确认全部通过
4. **输出审查报告**
   - 产出到 `work/reviews/review-task-{task_id}.md`
5. **更新任务状态**
   - 通过 → `done`，可合并到主线
   - 不通过 → `in_progress`，附上修改建议退回 Developer

## 产出
| 产物 | 路径 |
|---|---|
| 审查报告 | `work/reviews/review-task-{task_id}.md` |
| 任务状态 | 更新到 `work/tasks/task-NNN.md` |

## 质量门禁
- 至少检查：正确性、接口合规、代码质量、安全性
- 不通过时必须给出具体修改建议
- 测试覆盖率不低于 80%
